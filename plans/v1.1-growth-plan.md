# ScrollShame v1.1 — Growth Plan & Task Breakdown

> **Guiding principle:** Stop punishing users. Start making them proud of their chaos.  
> Every task below exists to serve one of three outcomes: **more shares**, **more conversions**, or **longer retention**.

# P0 — Critical Fixes (Ship This Week)

---

## TASK-01 · Rename "Shame" → "Chaos" Throughout

**Why:** "Shame Score" is punitive. "Chaos Score" is a badge. Identical data, completely different emotional response. This is the single highest-ROI change in the codebase.

**Files to change:**
- `src/lib/scoring.ts` — rename `getShameTitle()` labels, function rename optional but titles must change
- `src/report/main.tsx` — all text references to "Shame Score"
- `src/report/styles.css` — class names (`.shame-total`, `.total-score`, `.score-title`)
- `src/popup/main.tsx` — stat label "Shame"
- `src/popup/styles.css` — class `.stat-item:last-child`
- `dist/content.js` — any shame-labeled toasts

**Acceptance Criteria:**
- [ ] The word "shame" does not appear in any user-facing text in the report card, popup, or modal dialogs
- [ ] The large number on the report card is labeled **"Chaos Score"**
- [ ] The title below the score (e.g. "Tab Hoarder") is framed as a **rank/badge**, never a judgment
- [ ] `getShameTitle()` function is renamed `getChaosTitle()` (or aliased) and all 8 tiers use proud/absurd language — no tier should read as purely negative
- [ ] Build passes with 0 TypeScript errors

**Example tier rewrites:**

| Score | Old | New |
|:--|:--|:--|
| 90+ | Certified Internet Cryptid | **Grand Chaos Architect** |
| 80–89 | Certified Chaos Agent | **Certified Chaos Agent** *(keep — this one works)* |
| 65–79 | Tab Hoarder Supreme | **Tab Hoarder Supreme** *(keep)* |
| 50–64 | Casual Chaos Agent | **Chaos Enthusiast** |
| 35–49 | Mild Chaos Agent | **Chaotically Curious** |
| 20–34 | Scroll Apprentice | **Baby's First Chaos** |
| 10–19 | Mostly Innocent | **Suspiciously Calm** |
| 0–9 | Innocent Browser User | **We're Watching You** |

## TASK-03 · Add Week-Over-Week Score Delta

**Why:** "Your score went up 23 points" is a NEW shareable moment every single week. Without this, the report is identical in format every week — nothing new to share.

**Files to change:**
- `src/lib/scoring.ts` — add `calculateScoreDelta(current: number, previous: number)` and `getDeltaLabel(delta: number): string`
- `src/lib/storage.ts` — add `getLastWeekScore(): Promise<number | null>`, `saveWeekScore(score: number): Promise<void>`
- `src/report/main.tsx` — display delta badge below the Chaos Score

**New function signatures:**
```typescript
// scoring.ts
export function getDeltaLabel(delta: number): string
// returns: "▲ 18 points more chaotic" | "▼ 12 points less chaotic" | "Same chaos level as last week"

// storage.ts
export async function getLastWeekScore(): Promise<number | null>
export async function saveWeekScore(score: number, weekKey: string): Promise<void>
```

**Acceptance Criteria:**
- [ ] On first-ever report (no previous data): delta section is hidden
- [ ] Delta is calculated from the score saved the previous Monday
- [ ] Score is auto-saved to storage each time the report is opened
- [ ] If delta > 0: shows **`▲ {n} points — getting worse. impressive.`** in red
- [ ] If delta < 0: shows **`▼ {n} points — suspiciously reasonable this week`** in green
- [ ] If delta === 0: shows **`flat line. consistent chaos.`** in yellow
- [ ] Delta is visible in the report card (included in html2canvas screenshot)
- [ ] Delta does NOT affect the Chaos Score calculation — display only

## TASK-04 · Low-Score Celebration Mode

**Why:** The product currently only responds to chaos. If a user improves, nothing changes. Invisible improvement = no positive reinforcement = churn.

**Files to change:**
- `src/report/main.tsx` — conditional rendering when `chaosScore < 20`
- `src/lib/scoring.ts` — add `isCleanWeek(score: number): boolean`
- `src/lib/roast-pools.ts` — add `cleanWeek` pool per voice (3 lines each, same sarcastic tone)

**Acceptance Criteria:**
- [ ] When `chaosScore < 20`, the report card replaces the roast lines with **"clean week" copy** — same voice, but acknowledging the improvement suspiciously
- [ ] Clean week copy uses the same voice persona — the tone is still in-character (not sincere praise)
- [ ] Example for Therapist voice: *"Fewer than 20 tabs this week. I'm not going to say I'm proud of you. But I'm... noticing this."*
- [ ] The Chaos Score section shows the score with a different accent — use `--yellow` instead of `--red`
- [ ] A small "suspiciously clean week" badge appears next to the score title
- [ ] Share tweet for low scores gets different copy: *"My browser has nothing to say this week. That's either growth or deception."*

**Acceptance Criteria:**

- [ ] Streak resets if a week is missed (no report opened that week)

- [ ] Streak is displayed in popup as a 4th stat item: **"Streak"**

## TASK-13 · Improve Intervention Cooldown Persistence

**Why:** Intervention cooldowns reset on browser restart. This means a user who restarts Chrome gets intervention-bombed immediately.

**Files to change:**
- `dist/background.js` — persist `interventionCooldowns` to `chrome.storage.local`

**Acceptance Criteria:**
- [ ] Cooldown timestamps are stored in `chrome.storage.local` under key `intervention-cooldowns`
- [ ] On service worker restart, cooldowns are loaded from storage before any intervention check
- [ ] Cooldown format: `{ concerned: timestamp, zen: timestamp, random: timestamp }`
- [ ] If stored cooldown is older than its expiry window, it is treated as expired (not as zero)
- [ ] No user ever sees the same intervention type twice within its defined cooldown window, regardless of browser restarts
